> 从“复杂仿生闭环”退回到“可解释、可观测、可单元测试的离散神经网络模型”。

但是里面有几个地方要小心。尤其是：

```text
1/0/-1 信号可以用，但不能让整个系统都只有 1/0/-1。
```

否则模型会变得太离散，学习几乎没有“细微变化空间”。更合理的做法是：

```text
外部传播信号：1 / 0 / -1
内部状态变量：连续数值
突触权重：连续数值
阈值门控：连续数值
```

也就是：

```text
信号离散化
状态连续化
权重连续化
结构离散化
```

这样才既容易观察，又还能学习。

# 1. 总体评价

你的重构方向可以定义为：

> **双门控离散脉冲网络模型。**

它不是严格生物神经元模型，也不是传统人工神经网络，而是一种介于两者之间的工程抽象：

```text
节点 = 神经元
边 = 突触
输入/输出 = 离散信号
内部状态 = 膜电位/累积电荷
门控 = 阈值机制
连接 = 近邻生成 + 活动筛选 + 稳定性控制
学习 = 权重更新 + 连接重构
```

这个路线比你原来的项目更适合当前阶段，因为它更容易回答：

```text
这个节点为什么输出？
这条边为什么增强？
这个连接为什么断开？
这个网络是否真的学到了映射？
```

而不是一开始就被环境、运动、奖励、STDP、稳态、固化同时干扰。

# 2. 第一部分：重塑神经元，双门控是合理的

你提出两个门控：

```text
门控 1：树突与轴突之间的阈值阀门
门控 2：轴突统一信号阀门
```

这个想法可以保留，但我建议重新命名成更清楚的计算结构：

```text
门控 1：树突整合门 / dendritic gate
门控 2：轴突输出门 / axonal gate
```

更准确地说，一个神经元可以这样抽象：

```text
输入突触
→ 树突加权整合
→ 树突门控
→ 胞体状态累积
→ 轴突门控
→ 输出信号
```

也就是：

```text
synaptic inputs
→ dendritic sum
→ dendritic gate
→ soma potential
→ axon gate
→ output signal
```

---

## 2.1 推荐的神经元状态

每个神经元不要只保存 1/0/-1。它至少应该有这些状态：

```ts
type Neuron = {
  id: number;

  // 空间位置
  x: number;
  y: number;

  // 神经元类型
  role: "sensory" | "interneuron" | "output";
  kind: "excitatory" | "inhibitory" | "mixed"; // V0 可先 mixed，后面建议固定

  // 固定输入/输出槽位
  inputSlots: SynapseId[];
  outputSlots: SynapseId[];
  maxInputs: number;
  maxOutputs: number;

  // 内部连续状态
  dendritePotential: number; // 树突整合值
  somaPotential: number;     // 胞体/膜电位
  axonSignal: -1 | 0 | 1;    // 对外输出信号

  // 双门控阈值
  dendriteThreshold: number;
  axonThreshold: number;

  // 动态稳定机制
  leak: number;              // 衰减
  refractory: number;        // 不应期
  fatigue: number;           // 疲劳/输出抑制
  targetActivity: number;    // 目标活跃率
  activityEMA: number;       // 平均活动率
};
```

重点是：

```text
对外信号可以是 1/0/-1；
但内部必须有 somaPotential 这种连续变量。
```

否则神经元没有“接近阈值但还没放电”的状态，网络会非常粗糙。

---

## 2.2 双门控怎么计算？

我建议每个时间步这样算：

### 第一步：突触输入

```text
rawInput = Σ(inputSignal_j × weight_j)
```

其中：

```text
inputSignal_j ∈ {-1, 0, 1}
weight_j 是连续数值
```

例如：

```text
rawInput = 1×0.4 + 1×0.2 + (-1)×0.7 = -0.1
```

---

### 第二步：树突门控

树突门控不是直接放电，而是判断“输入是否值得进入胞体”。

```text
if |rawInput| < dendriteThreshold:
    dendriteOutput = 0
else:
    dendriteOutput = rawInput
```

也可以压缩成离散信号：

```text
if rawInput > dendriteThreshold:
    dendriteOutput = +1
elif rawInput < -dendriteThreshold:
    dendriteOutput = -1
else:
    dendriteOutput = 0
```

我更建议第一版用半连续：

```text
dendriteOutput = rawInput 或 0
```

不要太早完全离散化。

---

### 第三步：胞体状态累积

```text
somaPotential = somaPotential * leak + dendriteOutput
```

这一步很重要。

因为它允许神经元有“时间累积”：

```text
一次刺激不够
连续几次刺激够了
然后输出
```

这比纯组合逻辑更像神经元。

---

### 第四步：轴突输出门控

```text
if refractory > 0:
    output = 0

else if somaPotential > axonThreshold:
    output = +1
    somaPotential = 0
    refractory = N

else if somaPotential < -axonThreshold:
    output = -1
    somaPotential = 0
    refractory = N

else:
    output = 0
```

这就是你的第二个“轴突统一信号阀门”。

# 3. 第二部分：1/0/-1 信号可以用，但要明确它到底代表什么

你说三档信号：

```text
1 = 兴奋
0 = 无信号
-1 = 抑制
```

这个可以作为工程简化，但要避免一个生物学和建模上的坑：

> 真实神经元通常不是“这次发 +1，下次发 -1”。更多是神经元发放 spike，而突触类型决定它对下游是兴奋还是抑制。

也就是说，真实逻辑更像：

```text
神经元输出：spike / no spike
突触决定：兴奋性效果 or 抑制性效果
```

而不是：

```text
神经元自己每次输出 +1 或 -1
```

---

## 3.1 我建议的折中方案

你可以保留三档信号，但把它定义成：

> **突触后效应信号，而不是单纯轴突原始信号。**

也就是：

```text
pre neuron output = 0 / 1
synapse sign = +1 / -1
post input effect = preOutput × synapseSign
```

如果你坚持让神经元输出 -1，也可以，但要注意它会让模型变成“每个神经元都既能兴奋又能抑制”，这会降低结构可解释性。

更稳的 V0 设计是：

```text
神经元输出：0 / 1
突触效果：+ / -
最终输入：-1 / 0 / +1
```

例如：

```text
兴奋神经元放电 → 下游 +1
抑制神经元放电 → 下游 -1
不放电 → 0
```

这样既保留你的三档输入，又更接近神经系统。

# 4. 第三部分：固定数量输入/输出是好设计

你说：

> 一个神经元暂时只有固定数量的输入和输出。

这个非常合理。

因为之前那种随机连接如果不受控，很容易导致：

```text
某些节点成为超级节点
某些节点永远孤立
连接数量膨胀
评估困难
可视化困难
```

固定槽位可以让结构稳定很多：

```ts
type Neuron = {
  maxInputs: number;
  maxOutputs: number;
  inputSlots: Array<SynapseId | null>;
  outputSlots: Array<SynapseId | null>;
};
```

第一版建议：

```text
sensory neuron:
  maxInputs = 0
  maxOutputs = 4

interneuron:
  maxInputs = 4~8
  maxOutputs = 4~8

output neuron:
  maxInputs = 8
  maxOutputs = 0
```

不要一开始给太多连接。

建议第一版总规模：

```text
感觉神经元：8
中间神经元：24
输出神经元：4
总计：36
```

这比 64 还小，更适合重构验证。

# 5. 第四部分：连接形成机制，你的方向对，但需要分成三类规则

你提出：

```text
就近连接机制
阈值断开机制
防止短期重连接的标记
长期稳定性的双加权重
```

这些都合理，但必须分层。否则又会变成复杂迷宫。

我建议连接系统分成：

```text
连接生成规则
连接维护规则
连接断开规则
```

---

## 5.1 连接生成：近邻 + 空槽 + 兼容性

不要让任意神经元随便连。

每条新连接必须满足：

```text
1. pre 有空 output slot
2. post 有空 input slot
3. 距离在可连接范围内
4. pre/post 层级方向合理
5. 最近没有被断开标记禁止
```

例如：

```text
sensory → interneuron
interneuron → interneuron
interneuron → output
```

暂时不要允许：

```text
output → sensory
output → output
```

第一版可以禁止回路。
等前馈网络稳定后，再加入 recurrent connections。

---

## 5.2 连接得分

连接不是随机生成，而是有分数：

```text
connectScore =
    距离分
  + 共活动分
  + 空槽优先分
  - 重连惩罚
  - 过度连接惩罚
```

例如：

```text
distanceScore = exp(-distance / lambda)
coActivityScore = pre.activityEMA * post.activityEMA
cooldownPenalty = recentlyDisconnected ? 1 : 0
```

最终：

```text
if connectScore > connectThreshold:
    create synapse
```

---

## 5.3 阈值断开

断开不要只看当前权重小。

应该看：

```text
长期弱
长期不用
长期贡献负
```

例如：

```text
if abs(effectiveWeight) < minWeight
and useEMA < minUse
and age > minAge:
    disconnect
```

也就是说，至少满足三个条件：

```text
权重弱
使用少
不是刚出生连接
```

否则容易出现：

```text
刚连上 → 还没来得及用 → 被删
```

---

## 5.4 防止短期重连接：tombstone 标记

你的“标记”想法是对的。

断开后不要马上允许同一对节点重连。

可以设置：

```ts
type DisconnectionMark = {
  pre: number;
  post: number;
  untilTime: number;
  reason: "weak" | "harmful" | "overflow";
};
```

连接生成时检查：

```text
if tombstone(pre, post) still active:
    cannot reconnect
```

这可以防止：

```text
断开 → 立刻重连 → 又断开 → 震荡
```

第一版 cooldown 可以设为：

```text
5~20 秒模拟时间
```

# 6. 第五部分：双加权重是合理的，但要命名清楚

你说“双加权重”，我建议正式定义为：

```text
快速权重 fastWeight
稳定权重 stableWeight
```

或者：

```text
短期权重 labileWeight
长期权重 stableWeight
```

总权重：

```text
effectiveWeight = stableWeight + labileWeight
```

其中：

```text
labileWeight：快速变化，容易被奖励/惩罚影响
stableWeight：慢变化，需要长期一致才固化
```

---

## 6.1 为什么这很重要？

因为如果只有一个权重，会出现两个极端：

```text
学习快 → 容易崩
学习慢 → 学不到
```

双权重可以变成：

```text
labile 负责快速试错
stable 负责长期记忆
```

类似：

```text
先临时学会
反复有效后再固化
长期无效再慢慢忘记
```

---

## 6.2 推荐规则

### 快速更新

```text
labileWeight += learningRate * rewardSignal * eligibility
```

### 慢速固化

如果一条连接长期有效：

```text
if contributionEMA > stableThreshold
and useEMA > useThreshold:
    stableWeight += captureRate * labileWeight
    labileWeight *= (1 - captureRate)
```

### 慢速遗忘

```text
labileWeight *= labileDecay
stableWeight *= stableDecay
```

其中：

```text
labileDecay 快一些
stableDecay 慢很多
```

# 7. 第六部分：刺激信号规范是对的，但不要让负向刺激直接等于“负行为”

你说：

```text
营养因素 → 感觉神经传递正向信号
独立营养因素/有害因素 → 传递负向信号
```

这个作为第一版可以，但要注意一个坑：

> 如果毒物直接输入 -1，网络可能学成“毒物让系统沉默”，而不是“毒物触发避让动作”。

沉默不等于避让。

更合理的设计是：

```text
food sensory：检测到食物时输出 +1
toxin sensory：检测到毒物时也输出 +1
区别不在信号正负，而在通道身份
```

也就是说：

```text
S_food_left = 1
S_toxin_left = 1
```

然后网络学习：

```text
food_left → 左/右哪种运动
toxin_left → 相反运动或远离动作
```

如果你直接让：

```text
toxin = -1
```

那么它可能只是抑制整个网络，导致：

```text
看到毒物 → 不动
```

这未必是你想要的避害。

---

## 7.1 推荐的刺激编码

第一版使用“通道分离”，而不是“正负混合”。

```text
foodLeft:  0/1
foodRight: 0/1
toxinLeft: 0/1
toxinRight:0/1
bodySpeed: 0/1
turnLeft:  0/1
turnRight: 0/1
```

然后由突触决定兴奋/抑制。

这比：

```text
leftSignal = foodLeft - toxinLeft
```

更好调试。

因为你可以观察：

```text
到底是 food 通道没传进去？
还是 toxin 通道没传进去？
还是中间层没分化？
还是输出层没反应？
```

# 12. 一个完整时间步应该怎么跑？

```text
1. 清空每个神经元的 dendritePotential

2. 对每条突触：
   如果 pre.outputSignal != 0：
       effect = pre.outputSignal × synapse.sign × effectiveWeight
       post.dendritePotential += effect
       synapse.useEMA 更新

3. 每个神经元执行树突门控：
   如果 |dendritePotential| >= dendriteThreshold：
       somaPotential += dendritePotential
   否则：
       不进入胞体

4. somaPotential 衰减：
   somaPotential *= leak

5. 每个神经元执行轴突门控：
   如果 refractory > 0：
       outputSignal = 0
   否则如果 somaPotential > axonThreshold：
       outputSignal = +1
       refractory = refractoryMax
   否则如果 somaPotential < -axonThreshold：
       outputSignal = -1
       refractory = refractoryMax
   否则：
       outputSignal = 0

6. 根据目标/反馈计算 reward

7. 更新 eligibility

8. 更新 labileWeight

9. 稳定性调节：
   - 活跃率过高，提高阈值
   - 活跃率过低，降低阈值
   - 权重总量归一化

10. 连接维护：
   - 长期弱连接断开
   - tombstone 标记
   - 空槽尝试近邻连接
   - 长期有效连接固化
```

# 13. 你现在想法里的主要风险

## 风险 1：把抑制设计成负信号后，网络容易沉默

如果大量输入是 -1，很容易导致：

```text
somaPotential 长期为负
→ 输出很少
→ 学习事件减少
→ 网络沉默
```

解决：

```text
毒物不要直接等于 -1；
毒物最好是独立正向通道；
由后续突触结构决定避让。
```

---

## 风险 2：两个门控都太硬，信号穿不透

如果树突门和轴突门都很严格，会出现：

```text
输入进不来
进来后又出不去
```

所以第一版要让门控偏宽松：

```text
dendriteThreshold = 0.2 ~ 0.4
axonThreshold = 0.8 ~ 1.2
```

并且一开始不要加太多疲劳、不应期、复杂稳态。

---

## 风险 3：连接生成和断开同时启用，会造成结构震荡

如果你一开始就启用：

```text
新连接生成
弱连接断开
tombstone
双权重
固化
```

可能又会复杂化。

建议顺序：

```text
先固定连接
再学习权重
再断开连接
再生成连接
最后才固化
```

不要全部同时上。

---

## 风险 4：输出层没有明确评价，会学不到

你说“不一上来放到环境”，这是对的。
但离线任务必须有明确目标。

例如：

```text
input = foodLeft
target = outputLeft
```

否则没有判断标准。

# 15. 我的最终判断

你的新方案是合理的，但要做三个关键修正：

## 第一，1/0/-1 只能作为传播信号，不要作为全部内部状态

必须保留：

```text
somaPotential
weight
eligibility
activityEMA
```

这些连续变量。

---

## 第二，毒物/负刺激不建议直接编码成 -1

更建议：

```text
food 通道输出 1
toxin 通道也输出 1
```

区别由“通道身份 + 突触结构 + 奖惩学习”决定。

否则网络容易学成沉默，而不是避让。

---

## 第三，评测必须先脱离环境

这是你这次最重要的正确判断。

先让网络学会一个极小映射：

```text
foodLeft  → left
foodRight → right
toxinLeft → right
toxinRight→ left
```

等这个稳定了，再谈环境生存。
